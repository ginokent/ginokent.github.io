## Motivation

When trying to work with SVG in Rust, there are a few existing crates, but they all pull in too many dependencies for my taste. Fewer dependencies is always better.
None of them can evaluate SMIL animations (`<animate>`, etc.) either. Even if you only want path flattening or stroke conversion, those features are tightly coupled to a rendering pipeline and can't be used independently.

I knew exactly what I needed, so I wrote it myself.

- SVG parsing
- Bézier curve flattening
- Stroke-to-fill conversion
- SMIL animation evaluation

Everything uses only `std` — zero external crate dependencies.

[![ginokent/svg-rs](https://opengraph.githubassets.com/1/ginokent/svg-rs)](https://github.com/ginokent/svg-rs)

## Overview

svg-rs is a library that parses SVG into `PathSegment`s and handles flattening, stroke conversion, and SMIL evaluation. Each feature is an independent function, so you can use only what you need.

- **SVG Parsing** — Normalizes not only the `d` attribute of `<path>`, but also `<rect>`, `<circle>`, `<ellipse>`, `<line>`, `<polyline>`, and `<polygon>` into `PathSegment`s internally
- **Path Flattening** — Approximates quadratic/cubic Bézier curves and arcs into polylines with a specified tolerance
- **Stroke-to-Fill Conversion** — Converts strokes into fill paths, taking line width, line cap, and line join into account
- **SMIL Animation Evaluation** — Interprets `<animate>`, `<set>`, `<animateTransform>`, and `<animateMotion>` to compute attribute values at any given time

## Usage

### Parsing SVG

Pass SVG data to `parse()` and get back an `SvgDocument`.

```rust
use svg_rs::parse;

fn main() {
    let svg_data = br#"
        <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
            <rect x="10" y="10" width="80" height="80" fill="blue" />
            <circle cx="50" cy="50" r="30" fill="red" />
        </svg>
    "#;

    let doc = parse(svg_data).expect("SVG のパースに失敗");

    for element in &doc.elements {
        println!("{:?}", element);
    }
}
```

All shape elements are normalized into a list of `PathSegment`s, so downstream processing can be written uniformly.

### Path Flattening

Converts Bézier curves and arcs into sequences of line segments. This is needed for rasterization or polygon conversion.

```rust
use svg_rs::{parse, flatten};

fn main() {
    let svg_data = br#"
        <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
            <path d="M10,80 C40,10 65,10 95,80 S150,150 180,80" fill="none" stroke="black" />
        </svg>
    "#;

    let doc = parse(svg_data).expect("SVG のパースに失敗");
    let segments = &doc.elements[0].path_segments;

    // Smaller tolerance = higher precision but more points
    let polylines = flatten(segments, 0.5);

    for polyline in &polylines {
        println!("点の数: {}", polyline.len());
        for (x, y) in polyline {
            println!("  ({}, {})", x, y);
        }
    }
}
```

Tolerance is the maximum error in pixels. A value of `0.5` is usually sufficient.

### Stroke-to-Fill Conversion

Converts strokes (lines) into fill paths. This simplifies renderer implementation by allowing strokes and fills to be handled uniformly.

```rust
use svg_rs::{parse, stroke_to_fill, StrokeStyle};

fn main() {
    let svg_data = br#"
        <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
            <path d="M10,50 L90,50" stroke="black" stroke-width="10"
                  stroke-linecap="round" stroke-linejoin="round" />
        </svg>
    "#;

    let doc = parse(svg_data).expect("SVG のパースに失敗");
    let segments = &doc.elements[0].path_segments;

    let style = StrokeStyle {
        width: 10.0,
        line_cap: svg_rs::LineCap::Round,
        line_join: svg_rs::LineJoin::Round,
        miter_limit: 4.0,
    };

    let fill_segments = stroke_to_fill(segments, &style);
    println!("変換後のセグメント数: {}", fill_segments.len());
}
```

### SMIL Animation Evaluation

Interprets SVG SMIL animation elements and returns attribute values at a specified elapsed time.

```rust
use svg_rs::{parse, evaluate};

fn main() {
    let svg_data = br#"
        <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
            <rect x="10" y="10" width="80" height="80" fill="blue">
                <animate attributeName="x" from="10" to="50" dur="2s" fill="freeze" />
            </rect>
        </svg>
    "#;

    let doc = parse(svg_data).expect("SVG のパースに失敗");
    let animation = &doc.elements[0].animations[0];

    for t in [0.0, 1.0, 2.0] {
        if let Some(value) = evaluate(animation, t) {
            println!("t={:.1}s: {:?}", t, value);
        }
    }
}
```

Supported SMIL elements:

| Element | Example Attributes |
|------|-----------------|
| `<animate>` | `x`, `y`, `width`, `height`, `opacity`, `fill`, etc. |
| `<set>` | Any attribute (discrete value switching) |
| `<animateTransform>` | `translate`, `rotate`, `scale`, `skewX`, `skewY` |
| `<animateMotion>` | Movement along a `path` |

## CLI Tool (svg-cli)

A CLI tool is also included to access svg-rs features from the command line.

Rendering a static PNG:

```bash
svg-cli render input.svg -o output.png --width 512 --height 512
```

Generating an APNG from an SVG with SMIL animations:

```bash
svg-cli animate input.svg -o output.apng --fps 30 --duration 3.0
```

Main options:

| Option | Description |
|-----------|------|
| `--width`, `--height` | Output size (pixels) |
| `--fps` | Frame rate |
| `--duration` | Animation length (seconds) |
| `--background` | Background color (e.g., `#ffffff`) |
| `--tolerance` | Flattening tolerance (default: `0.5`) |

## Project Structure

```text
svg-rs/
├── src/
│   ├── lib.rs          # Public API (parse, flatten, stroke_to_fill, evaluate)
│   ├── parser/
│   │   ├── mod.rs      # XML/SVG parser
│   │   ├── path.rs     # Path data (d attribute) parser
│   │   └── shape.rs    # Shape element → PathSegment conversion
│   ├── flatten.rs      # Bézier curve and arc flattening
│   ├── stroke.rs       # Stroke → fill conversion
│   ├── smil/
│   │   ├── mod.rs      # SMIL animation evaluator
│   │   ├── timing.rs   # Timing calculations (dur, repeatCount, begin, end)
│   │   └── interpolate.rs  # Value interpolation (linear, discrete, spline)
│   └── cli/
│       └── main.rs     # CLI entry point
├── tests/
├── Cargo.toml
└── README.md
```

Each module is independent, so you can read `parser` or `flatten` in isolation and still understand what's going on.

## Key Features

**Fast builds.** With no external crates, a clean build finishes in seconds. CI is fast too.

**Works with Wasm.** Since it only uses `std`, it compiles to `wasm32-unknown-unknown` without any issues. You can use it as-is for things like rendering SVG with Wasm in the browser. `no_std` support is something I'd like to add eventually.

**No supply chain risk.** This goes without saying, but with zero dependencies, you don't have to worry about supply chain attacks.

## Limitations

Currently unsupported features:

- CSS `<style>` element styling
- `<text>` elements
- `<filter>` elements
- Some `<mask>` / `<clipPath>` features

These will be addressed as needed.

## Conclusion

I built a Rust library that handles SVG parsing, flattening, stroke conversion, and SMIL evaluation with zero dependencies. It's useful for things like building a lightweight SVG renderer from scratch or generating APNGs from SMIL animations.
