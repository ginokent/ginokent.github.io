# 設計仕様

## i18n ルーティング

### 方針

デフォルト言語（日本語）の URL には言語プレフィックスを付けない。非デフォルト言語（英語）のみプレフィックスを付ける。

| URL | 言語 |
|-----|------|
| `/about` | 日本語 |
| `/en/about` | 英語 |
| `/ja/about` | **404**（存在しない） |

### 実装

Astro の `i18n.routing.prefixDefaultLocale: false` を使用している。この設定では `/ja/...` のルートは生成されないため、デフォルト言語用のページと `[lang]` パラメータを使った多言語用のページを**2ファイル構成**で管理している。

```
src/pages/about.astro          # 日本語（デフォルト言語）
src/pages/[lang]/about.astro   # 英語など（非デフォルト言語）
```

この構成はサイト内の全ページ（index, posts, scraps, tags, about）に適用されている。

### 制約

- Astro 標準の i18n routing では「プレフィックスなし」と「`/ja` プレフィックスあり」の両方で同じページにアクセスする方法は提供されていない
- そのため、同じ内容のページを2ファイルで管理する必要があり、変更時は両方のファイルを更新する必要がある

### 2ファイル間のデータ共有について

2ファイル間で共通のデータ（例: about ページのリンク配列）を共通ファイルに切り出すことは意図的に行わない。各ページファイルは import パスや `getStaticPaths` の有無など構造が異なるためシンボリックリンクも使えず、共通化の対象はページ内のごく小さなデータに限られる。変更頻度が低いデータのために抽象化を導入するのは過剰であり、2箇所を同時に編集する方が単純で見通しが良い。データ量や変更頻度が増えた時点で再検討する。
